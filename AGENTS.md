# Migration instructions

## Verbatim content is a hard requirement

When migrating any page from `andrewdessler.com` into this repository:

- Preserve all user-facing source text verbatim. Do not rewrite, summarize, shorten, expand, polish, modernize, or correct it.
- Preserve wording, spelling, capitalization, punctuation, quotations, names, dates, paragraph order, headings, captions, and link labels exactly as they appear on the source page.
- Copy the complete page content. Do not omit text because it seems repetitive, outdated, unimportant, or too long.
- Preserve every source link and associate it with the same linked text. Update a URL only when Andrew explicitly supplies a replacement or explicitly requests a link change.
- If Andrew supplies newer factual text that conflicts with the Squarespace page, preserve Andrew's newer text and otherwise keep the source verbatim.
- Layout, responsive behavior, typography, colors, navigation structure, accessibility markup, and other presentation details may be redesigned as long as the visible source text is unchanged.
- New interface labels that are necessary for navigation or accessibility may be added, but they must not replace, paraphrase, or alter migrated content.

Before committing a migrated page, compare its rendered visible text and links against the source page. Resolve every unexplained difference. If exact preservation is impossible or the intended source is ambiguous, stop and ask Andrew rather than rewriting by assumption.
